# Model Context Protocol (MCP) Integration with Oracle Database

The Model Context Protocol (MCP) is an open standard that enables seamless communication between Artificial Intelligence (AI) applications and various data sources. Oracle provides robust support for MCP, allowing developers to connect Large Language Models (LLMs) to both the latest Oracle AI Databases and older versions of Oracle Database.

## Overview of Oracle MCP Solutions

Oracle offers two primary implementations of MCP servers to cater to different deployment scenarios and database versions:

| MCP Server Type | Target Database | Deployment | Key Advantage |
| :-------------- | :-------------- | :--------- | :------------ |
| **Oracle Autonomous AI Database MCP Server** | Oracle Autonomous AI Database (Serverless) | Server-side (OCI) | Native integration with OCI security and cloud-native workflows [1]. |
| **Oracle SQLcl MCP Server** | Any Oracle Database (11g, 12c, 19c, 21c, 23ai, 26ai) | Client-side (Local/Container) | Bridges modern AI capabilities to legacy and on-premises databases [2, 3]. |

## Oracle SQLcl MCP Server: Bridging the Gap

The Oracle SQLcl MCP Server is a versatile tool that allows AI applications to interact with any Oracle Database that SQLcl can connect to. This is particularly valuable for organizations using older versions of Oracle Database who wish to leverage modern AI capabilities without an immediate database upgrade [4].

### How it Works:
SQLcl acts as a bridge, running on the client machine or in a container. It translates natural language requests from an MCP-compatible AI client (like Claude Desktop) into SQL queries and scripts, which are then executed against the connected Oracle Database [3, 5].

### Core Tools Provided:
*   **run_sql**: Executes SQL queries generated by the LLM.
*   **describe_table**: Retrieves detailed schema information for specific tables.
*   **list_tables**: Provides a list of available tables within the connected schema.
*   **create_report**: Generates formatted reports based on database data.

## Integration with Older Oracle Database Versions

To integrate older versions of Oracle Database (e.g., 12c or 19c) with an MCP server, follow these general steps:

1.  **Install SQLcl**: Ensure you have the latest version of Oracle SQLcl (Release 25.2 or later) installed on your client machine [2].
2.  **Configure Connection**: Set up a standard database connection to your older Oracle instance using SQLcl's connection store.
3.  **Launch MCP Server**: Start SQLcl in MCP mode using the command: `sql /nolog mcp`.
4.  **Connect AI Client**: Configure your MCP-compatible AI application to communicate with the SQLcl MCP server process [6].

## Security and Auditing

Granting an LLM access to a database requires careful security considerations. Oracle provides several built-in mechanisms to ensure secure and auditable interactions:

*   **Least Privilege Access**: Always use a database user account with the minimum permissions necessary for the AI's tasks [2].
*   **Session Tracking**: The database populates `V$SESSION.MODULE` with the MCP client name and `V$SESSION.ACTION` with the LLM's name for easy identification [2].
*   **Activity Logging**: Every interaction and SQL execution is recorded in a dedicated table named `DBTOOLS$MCP_LOG` [2].
*   **Query Tagging**: All queries generated by the LLM through the SQLcl MCP Server include a identifying comment: `/* LLM in use ... */` [2].
